# Plan 04-01: Implement Follow-up Drain Consumer

## Objective

Implement the consume-side of the `pendingFollowUps` queue in `GroupQueue`. When an agent turn completes, any messages that arrived during that turn (pushed to `state.pendingFollowUps` by `sendMessage()`) must be sent to Claude via `state.session.send()` and their responses routed to the user. Currently this array is never drained — messages are silently discarded.

  <action>
    In `src/group-queue.ts`:

    1. Add `drainFollowUpsFn` private field to GroupQueue class (type: `((groupJid: string, text: string) => Promise<void>) | null = null`).

    2. Add `setDrainFollowUpsFn(fn: (groupJid: string, text: string) => Promise<void>): void` public setter.

    3. Add private async method `drainFollowUps(groupJid: string, state: GroupState): Promise<void>`:
       ```typescript
       private async drainFollowUps(groupJid: string, state: GroupState): Promise<void> {
         while (state.pendingFollowUps.length > 0 && state.session) {
           const msgs = state.pendingFollowUps.splice(0);
           const combined = msgs.join('\n\n');
           logger.debug({ groupJid, msgCount: msgs.length }, 'Draining follow-up messages');
           try {
             await state.session.send(combined);
             for await (const msg of state.session.stream()) {
               if (msg.type === 'assistant') {
                 const text = (
                   msg.message.content as Array<{ type: string; text?: string }>
                 )
                   .filter((b): b is { type: 'text'; text: string } => b.type === 'text')
                   .map((b) => b.text)
                   .join('');
                 if (text && this.drainFollowUpsFn) {
                   await this.drainFollowUpsFn(groupJid, text);
                 }
               }
               if (msg.type === 'result') break;
             }
           } catch (err) {
             logger.error({ groupJid, err }, 'Error draining follow-up, stopping drain');
             break;
           }
         }
       }
       ```

    4. In `runForGroup()`, inside the `try` block, after `await this.processMessagesFn(groupJid)` returns (both success and retry paths), add:
       ```typescript
       await this.drainFollowUps(groupJid, state);
       ```
       This must be placed AFTER the `if (success) / else scheduleRetry` block but BEFORE the `finally` block. The `state.session` is still valid at this point — `finally` nulls it later.

    NOTE: `drainFollowUps` must NOT be called in the `catch` block of `runForGroup` — if `processMessagesFn` throws, the session state is uncertain. Only drain on clean exits.
