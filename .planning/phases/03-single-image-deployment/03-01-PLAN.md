  <action>
创建两个文件：

**Dockerfile**（两阶段 multi-stage build）：

Stage 1 (builder):
- Base: `FROM node:20-slim AS builder`
- WORKDIR /app
- 安装编译工具（仅 builder stage）：`apt-get install -y --no-install-recommends python3 g++ build-essential`，之后 `rm -rf /var/lib/apt/lists/*`
- COPY package.json package-lock.json ./
- `RUN npm ci --omit=dev`（在容器内编译 better-sqlite3，确保 ABI 匹配 runner stage）
- 注意：builder stage 不设置 NODE_ENV=production，明确用 `--omit=dev` 控制
- COPY tsconfig.json ./
- COPY src ./src
- `RUN npm run build`（产出 dist/）

Stage 2 (runner):
- Base: `FROM node:20-slim AS runner`
- WORKDIR /app
- 创建持久化目录（bind mount 会覆盖，但容器内需要占位）：`RUN mkdir -p /app/store /app/data /app/groups/main`
- COPY --from=builder /app/node_modules ./node_modules
- COPY --from=builder /app/dist ./dist
- COPY package.json ./
- 权限：`RUN chown -R node:node /app`
- 切换用户：`USER node`
- 设置环境：`ENV NODE_ENV=production`
- 启动命令：`CMD ["node", "dist/index.js"]`

不添加 EXPOSE（服务不监听端口）。
不添加 HEALTHCHECK（WhatsApp 连接状态难以用 HTTP 端点检测）。
